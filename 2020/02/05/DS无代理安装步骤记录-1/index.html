NSX + 趋势 Deep Security 无代理防病毒部署手册

原创 Spark go nsx很可爱的 2018-08-28

在NSX从入门到精通(7)：NSX 微分段架构、组件及实践 Part2中，简单介绍了第三方安全产品如何借助NSX安全组及安全策略与 NSX 实现集成，因篇幅原因只简单提了 NSX 部分的配置，这篇文章便是一个完整的无代理防病毒部署过程，使用趋势最新版本的无代理防病毒软件 Deep Security v11，其他同类无代理防病毒也可以参考此文章，部署过程差别不大。

00

—

目录/流程

01

—

基础资源需求

在部署 Deep Security 无代理防病毒时，需要准备以下安装介质：

  • NSX manage**r**： 提供安全接口，对接趋势安全平台；

  • Deep Security Manager：趋势 Deep Security 平台的控制节点，与 NSX 及vCenter 对接；

  • 数据库：DS Manager需要使用，支持 SQL server、Oracle DB、Postgres，具体兼容情况请参见详细的官方部署文档；

  • Deep Security Agent：部署在 Windows/Linux 上的代理程序，能够提供杀毒、防火墙、Relay 等功能。趋势平台必须至少有一台服务器与互联网连接，做 Relay server 负责更新工作，此虚拟机必须安装 Agent；

  • Deep Security ESX Appliance：执行无代理防病毒的 SVM 虚拟机安装包；

  • Deep Security Notifier（可选）：可以在 Windows 虚拟机下提供安全事件弹窗。

未来需要至少5台虚拟机，请预留好 IP 地址：

1、NSX manager 虚拟机， 通过 ova 在 vCenter 中导入；

2、Deep Security Manager 虚拟机，可以是 Windows 或者 Linux，需要提前准备好 Deep Security Manager 安装文件；

3、数据库虚拟机，请根据 Deep Security Manager 需求安装合适的数据库；

4、更新 Relay 虚拟机，需要连接互联网，安装 agent；

5、测试虚拟机，建议使用 Windows 虚拟机进行测试。

02

—

兼容性查询

在使用无代理防病毒时，需要做的第一件事是检查vSphere版本、NSX 版本与 Deep Security 三者间的兼容性，可以通过以下链接查询（复制后在浏览器打开）：

https://www.vmware.com/resources/compatibility/search.php?deviceCategory=security

查询结果如下，可以使用 NSX 6.4.1 的版本，使用 Deep Security 11.0版本。

03

—

下载 Deep Security 安装文件

安装文件下载链接：

https://help.deepsecurity.trendmicro.com/software.html?regs=NABU&clk=latest&clkval=5033&lang_loc=1

打开链接后选择需要的软件包进行下载：

04

—

趋势 Deep Security 的安装

1、安装 Deep Security Manager

前置条件：已经安装完数据库，并创建名为 dsm 的数据库。

打开 Manager 安装文件。

选择语言，目前新版 v11 只有英文。

接受许可证。

自动执行预检查。

选择安装路径。

设置数据库连接信息。

点击 “Start Readiness Check” 进行安装环境检查。

立即输入许可证或者稍后输入。

根据现场情况填写 Manager IP 或者域名。

设置管理员用户名及密码。

更新选项。

是否为 Manager 虚拟机启用 Relay-enabled Agent 用于更新服务。

是否加入 SMART Protection Network。

确定所有设定无问题后，点击 Install 开始安装。

安装结束。

打开浏览器测试是否可以正常登陆。

2、安装 Relay Server（更新中继服务器）

前置条件：Relay Server 虚拟机已经准备就绪，可以正常访问互联网，且可以和 Deep Security Manager 通信。

下载 For Windows Agent 的 ZIP 包后解压，打开安装程序 。

登陆 Deep Security manager，进入 Computer 下，点击 Add。

输入 Relay 虚拟机的 IP 。

添加完成后状态为 Managed 。

进入 Administration > Relay Management，新增 Relay Group。

选择刚才添加的节点，点击 Enable Relay and Add to Group。

添加成功。

3、导入 ESX Appliance

打开 Administration > Updates > Software > Local，导入已经下载好的 ESX Appliance（也可以在 Relay Sever 设置好后通过 Administration > Updates > Software > Download 下载）。

05

—

NSX 的安装

下面内容 IP 配置会与其他截图不匹配，请务必按照现场情况及计划进行 IP 等的配置。

1、部署 NSX Manager Appliance

导入 OVA 文件。

设置虚拟机位置。

选择运行主机。

验证模板信息。

选择存储。

选择 NSX Manager 的管理端口组，与 vCenter 通信时使用。

对 NSX Manager 进行自定义，这里设置 admin 密码和 CLI 模式的密码。

展开 Network Properties，填写 NSX Manager 主机名和配置管理的 IPv4 地址与vCenter 通信使用。

展开 DNS 和 Services Configuration，配置 DNS 和域名，配置 NTP 服务器信息

注意：NSX 的时间同步必须要与 vCenter 和 esxi 主机一致，时间敏感！

确认配置信息，并完成 Appliance 部署。

2、设置 NSX Manager 与 vCenter 关联。

登陆 https://10.10.50.10，使用 admin 和设置好的密码登陆 NSX Manager 管理页面。

（可选）选择 Manager Appliance Settings，配置 IP、NTP、主机名、DNS 和备份恢复等信息。

选择 Manage vCenter Registration，配置 Lookup Service（PSC的登陆凭据）和vCenter 登陆凭据。

显示绿灯后表示 NSX Manager 与 vCenter 连接成功。

稍等一会，重新登陆vCenter主页（使用NSX注册vCenter SSO和vCetner服务的权限，如：administrator@vsphere.local），会出现 Networking & Security 控件和按钮，点击进入NSX管理页面。

06

—

NSX 与 Deep Security 集成与配置

1、**在 Deep Security manager 中注册 vCenter**

打开 Deep Security Manager，点击 Computers>Add>Add VMware vCenter。

填写 vCenter 登陆凭据。

信任证书。

填写 NSX manager 登陆凭据。

添加完成。

加载完成后可以看见所有虚拟机。

登陆 vCenter，打开网络与安全>服务定义，检查 Trend Micro Deep Security 在列表中。

2、配置 SVM 虚拟机的存储路径与网络端口组

下图为一个标准的 NSX + 第三方无代理防病毒架构图，图中最底层的 MUX 存在于 ESXI 主机内部，会自动分配一个私有 IP 地址与 GI-SVM 及合作伙伴 SVM 内部通讯。GI-SVM 及 Partner-SVM 均有一个对外的接口分别和 NSX manager 以及合作伙伴的控制端连接。

本章节便是为 GI-SVM 和 Partner-SVM 预先配置存储位置和网络配置。

登陆 vCenter，选择集群中的每台主机，打开配置>代理虚拟机设置>编辑。

选择合适的数据存储（可以使用本地数据存储）和端口组（所在 VLAN 必须和 DS Manager 网络通）。

按照同样方式设置第二台主机 。

3、安装 Guest Introspection

进入 NSX 网络与安全配置，点击安装和升级>服务部署>添加。

选择在哪些集群上部署 GI 虚拟机。

数据存储和网络选择“已在主机上指定”。

IP 分配选择 IP 地址池（按照现场环境，也可以选择 DHCP）。

配置 IP 池，地址范围必须≥主机数量。

等待安装完成，确保安装状态和服务状态（在安装成功后需要等待数分钟）均为绿色。

4、安装 Deep Security SVM

进入 NSX 网络与安全配置，点击安装和升级>服务部署>添加。

选择 Trend Micro Deep Security。

选择在哪些集群上部署 Deep Security SVM 虚拟机。

配置 Deep Security SVM 地址池

等待部署完成。

5、设置导流规则

在以上所有部署完成后，需要在 NSX 中设置安全组和安全策略。

安全组包含所有需要执行防病毒的虚拟机，安全策略中设置导流规则，最后再做安全组和安全策略的关联。

进入 NSX 网络与安全配置，点击安全性>服务编排>安全组>添加。

输入安全组名称。

使用动态或者静态规则，将需要执行防病毒的虚拟机加入该组，点击完成。

点击安全策略>添加。

输入安全策略名称。

在 Guest Introspection 服务中，点击添加。

应用到 Trend Micro Deep Security，状况为已启用。

点击完成。

等待创建完成。

选择刚才创建的安全策略，点击应用，将策略应用于刚才创建的安全组。

6、为虚拟机安装 VMware Tools

打开 vCenter，选择测试虚拟机，点击安装 VMware Tools。

打开虚拟机 Console，打开 VMware Tools 安装光盘。

选择完整安装或者自定义安装。

找到 VMCI 驱动程序，NSX 文件自检支持，选择将此功能安装在本地硬盘。

等待重启完成。

7、防病毒策略下发

在NSX端配置完导流规则后，进入 DS manager>计算机，选中需要执行防病毒的虚拟机，右键点击 Action>Assign Policy。

选择 Assign Policy>Windows Sever。

等待状态变为 Managed。

8、防病毒测试

通过远程桌面，将预先下载好的病毒样本拖放到测试虚拟机中，会发现文件无法打开，已被静默删除。

安装 Deep Security Notifier。

安装完成后，系统托盘会有趋势的图标。

再执行病毒样本的复制，系统会出现通知。

回到 DS manager>computer，双击测试虚拟机。

点击 Anti-Malware>Anti-Malware Events，查看此计算机的防病毒事件，可以看到刚才多次测试结果，病毒样本已被隔离。

至此，整个环境已经搭建完成。
